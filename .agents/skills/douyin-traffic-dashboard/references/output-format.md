# Output Format

本文件描述 `douyin-traffic-dashboard` 技能输出结果的格式规范。

## 整体结构

输出遵循 **结论 -> 证据 -> 建议** 三段式结构，确保回答简洁有逻辑。

```
结论：<一句话总结，当前抖音流量分配的核心发现>

证据：
- <分类1> 流量占比 <X>%，热度排名 #<N>，趋势 <上升/平稳/下降>
- <分类2> 流量占比 <X>%，热度排名 #<N>，趋势 <上升/平稳/下降>
- ...

建议：
1. <针对流量增长赛道的创作建议>
2. <针对新兴机会赛道的建议>
3. <针对高竞争赛道的差异化建议>
```

## 核心返回字段

API 返回的流量分布数据字段（按 `traffic-distribution` 聚合模式）：

| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 内容分类名称/标签 | `美食`, `生活方式`, `科技` |
| `rank` | 当前热度排名 | `1` |
| `rank_diff` | 排名变化幅度（正数表示上升，负数表示下降） | `5` |
| `tag` | 内容所属分类 | `美食` |
| `hot_value` | 热度值（可选） | `123456` |
| `traffic_share` | 流量占比（%，由 `traffic-distribution` 聚合模式计算得出） | `28` |
| `trend` | 流量趋势（上升/平稳/下降，由 `rank_diff` 推导） | `上升` |

## 字段说明

### 热度排名（rank）

内容分类在抖音热榜中的绝对排名位置。数字越小，热度越高。

- 排名前 3：超高流量分类，流量巨大但竞争极为激烈
- 排名 4-10：高流量分类，流量充沛
- 排名 11-20：上升中分类，有流量空间
- 排名 20+：新兴分类，先发优势明显

### 变化幅度（rank_diff）

相比上一统计周期的排名变化，正数表示排名上升，负数表示排名下降。

- `+10` 及以上：爆发期，该分类正在快速获得流量倾斜
- `+5` 至 `+9`：上升期，流量增长稳定，值得关注
- `+1` 至 `+4`：缓慢上升，可作为备选赛道
- 0：流量平稳，该分类处于成熟稳定状态
- 负数：流量衰退，该分类正在失去平台流量支持

### 流量占比（traffic_share）

该分类占抖音整体内容消费流量的百分比，是判断赛道规模的核心指标。

- 30% 及以上：超大规模赛道，流量红利期已过
- 20%-29%：大规模赛道，流量充沛但竞争激烈
- 10%-19%：中等规模赛道，有增长空间
- 10% 以下：新兴/垂直赛道，先发优势明显

### 趋势判断（trend）

综合 `rank_diff` 和 `traffic_share` 推导的流量趋势判断：

- **上升**：`rank_diff >= +5`，该分类正在快速崛起
- **平稳**：`rank_diff` 在 -4 到 +4 之间，流量稳定
- **下降**：`rank_diff <= -5`，该分类流量正在流失

## 输出示例

### 成功响应（结论 -> 证据 -> 建议）

```
结论：当前抖音流量明显向美食、情感和生活方式三大分类集中，美食类流量占比持续扩大。

证据：
- 美食 流量占比 28%，热度排名 #1，趋势 上升
- 情感 流量占比 22%，热度排名 #2，趋势 平稳
- 生活方式 流量占比 18%，热度排名 #3，趋势 上升
- 科技 流量占比 12%，热度排名 #4，趋势 上升
- 娱乐 流量占比 10%，热度排名 #5，趋势 下降

建议：
1. 美食类创作者可继续深耕，但建议寻找差异化角度（如地方特色、小众菜系）
2. 情感类内容流量稳定，适合情感类账号持续产出
3. 生活方式赛道正在获得流量倾斜，新账号可优先考虑入局
4. 科技类门槛较高但竞争小，适合有专业背景的创作者
```

### 无结果响应

```
结论：当前未获取到抖音流量分配数据，可能 API 暂时不可用。

建议：
1. 检查 AISKILLS_API_KEY 和 AISKILLS_TENANT_ID 环境变量是否正确配置
2. 稍后再试，热点数据更新有一定延迟
3. 直接访问抖音热榜查看实时数据
```
